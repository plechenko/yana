# Contributing

Contributions to YANA and YANA Modules are welcome.

## Ways to Contribute

- Submit bug reports and feature requests via [GitHub Issues](https://github.com/oops-42/yana/issues)
- Propose code improvements as [Pull Requests](https://github.com/oops-42/yana/pulls)
- Contribute reusable modules to the [YANA Modules Repository](https://github.com/oops-42/yana-modules)
- Share and showcase your experience with YANA

## Pull Request Guidelines

1. Fork the repository and create a branch from `main`.
2. Write or update tests for any changed behavior using the [Testing Framework](testing.md).
3. Ensure all tests pass:
   ```bash
   ./yana-test.sh
   ```
4. Keep changes focused and minimal. One concern per PR.
5. Open a pull request with a clear description of the change and why it is needed.

## Code of Conduct

All contributors are expected to follow the [Contributor Covenant Code of Conduct](https://www.contributor-covenant.org/version/2/0/code_of_conduct/).
